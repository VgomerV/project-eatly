# KODA.md — Инструкции для AI-ассистента

## Обзор проекта

**Название:** Eatly  
**Тип:** Веб-приложение (лендинг) для сервиса доставки еды  
**Сборщик:** Vite  
**Технологии:** HTML5, CSS3, JavaScript (минимальный)

Проект представляет собой маркетинговый лендинг сервиса доставки еды с несколькими секциями: главный экран (hero), особенности сервиса, мобильное приложение, популярные рестораны, блюда, панель управления заказами, отзывы клиентов, скидки и подвал.

## Команды для работы с проектом

### Установка зависимостей
```bash
npm install
```

### Запуск dev-сервера
```bash
npm run dev
```
Запускает локальный сервер с горячей перезагрузкой (HMR).

### Сборка проекта
```bash
npm run build
```
Создаёт оптимизированную сборку в папке `dist`.

### Превью сборки
```bash
npm run preview
```
Запускает локальный сервер для просмотра готовой сборки.

## Архитектура и стиль кодирования

### HTML
- Семантическая вёрстка с использованием тегов `<header>`, `<main>`, `<section>`, `<nav>`, `<footer>`
- Классы по методологии БЭМ (например: `menu__link`, `hero__title`, `button--filled`)
- Использование SVG-иконок прямо в HTML

### CSS
- Организация стилей: один файл на каждый компонент/секцию
- Главный файл `style.css` импортирует все остальные
- Использование CSS-переменных для цветов (проверить в `main.css`)
- Адаптивная вёрстка (проверить наличие медиа-запросов)
- Градиенты, тени, анимации для визуальных эффектов

### JavaScript
- На данный момент JS практически не используется
- Интерактивность реализована через CSS (checkbox-hack для мобильного меню)

## Рекомендации по доработке

1. **Добавить TypeScript** — проект может быть переведён на TS для типобезопасности
2. **Вынести переиспользуемые компоненты** — кнопки, карточки, заголовки
3. **Добавить документацию** — создать README.md с инструкциями
4. **Настроить линтер и форматер** — добавить ESLint/Prettier для CSS
5. **Оптимизировать изображения** — использовать WebP формат

## Особенности реализации

- Мобильное меню реализовано через checkbox-hack (без JavaScript)
- Используются внешние шрифты (подключены через Google Fonts или локально)
- Присутствует SVG-графика для декоративных элементов
- Цветовая схема: фиолетовые/синие акценты на тёмном фоне
